# rust-IME 使用手册

这是一个专为 Linux 设计的高性能、现代化中文输入法引擎。采用 Rust 编写，深度适配 Wayland/COSMIC 桌面环境，提供极致的丝滑盲打体验。

---

## 💡 设计哲学与初衷 (Philosophy)

### 1. 为什么要做这个输入法？
拼音输入法最大的痛点在于 **“重码选择”**。即便全拼输入非常自然，但面对“里、力、理、利”等大量同音字时，选词过程极大地打断了思维流。

更深层的动力源于 **Linux Wayland 生态的输入困局**：
在现代 Linux（如 COSMIC 桌面）中，许多基于 Rust `iced` 库开发的原生应用对 `Fcitx5` 或 `IBus` 的支持极差。即使是系统自带的终端和文件管理器，也经常出现无法呼出输入法的情况。这通常是因为 Wayland 的 `text-input` 协议实现不统一导致的。

**rust-IME 的“降维打击”：**
- **原生通用性**：既然应用能接收英文、能接收粘贴，那我们就把自己伪装成一个**真实键盘**。通过物理级的按键模拟 + 剪贴板自动中转方案，绕过复杂的 Wayland 协议逻辑。
- **环境要求**：本项目高度依赖系统剪贴板服务（Clipboard）。这意味着在具备粘贴能力的 Wayland/X11 应用（包括 COSMIC 终端、Firefox 等）中表现极佳，但目前**无法在不具备剪贴板环境的 TTY 原生模式下工作**。
- **英文辅码 (English Auxiliary)**：调用本能的英文知识实现**零重码盲打**。例如：输入 `niM`（M 代表英文单词 Mud）。

### 2. 行业锐评 (Hot Takes)
- **对比五笔**：学习曲线极其陡峭，且容易忘记拆解规则。
- **对比双拼**：只解决了“输入快”，没解决“选词慢”。
- **结论**：**全拼 + 英文辅码 + 简拼** 才是速度与直觉的完美平衡。

## 🛠 核心功能特性

### 1. “分段锁定”长句输入 (Space Anchor)
借鉴日本输入法的设计精髓，引入**空格锚定逻辑**：
- **用法**：输入拼音时按下 `Space`（空格），当前段落的首选词将被**物理锁定**，不再受后续输入的影响而变动。
- **优势**：允许用户通过空格手动控制语义边界，实现超长、稳定的句子录入。

### 2. 四声调过滤与智能纠错 (Tones)
利用数字键快速锁定声调，消除 80% 的重码：
- `7`: 一声 (ā) | `8`: 二声 (á) | `9`: 三声 (ǎ) | `0`: 四声 (à)
- **用法**：输入 `ni9` -> 直接过滤出“你”、“拟”等三声字。

### 3. 卡片式极客 UI (Modern Card UI)
除了传统的横向候选框，新增**独立悬浮的卡片式界面**：
- **视觉风格**：Manjaro 绿配色，高透明度卡片感，完美适配暗黑模式。
- **独立控制**：支持通过快捷键随时在传统、卡片、隐藏三种模式间自由切换。

### 4. 高性能二进制架构
- **0 秒启动**：基于 `memmap2` 内存映射，加载均为微秒级。
- **自动增量编译**：启动时自动检测 `dicts/` 目录下的 JSON/YAML 源文件变动，实现无感知的二进制索引更新。

## ⌨️ 默认快捷键 (Default Hotkeys)

| 快捷键 | 功能描述 |
| :--- | :--- |
| **`Caps Lock`** | 切换中 / 英文输入模式 |
| **`Ctrl + Space`** | 切换中 / 英文输入模式 (备选) |
| **`Enter`** | 确认首选词上屏 (Confirm Selection) |
| **`Space`** | 输入手动分词符 (Syllable Separator) |
| **`Delete`** | 物理级一键清空输入缓冲区 |
| **`Tab`** | 候选词向下滚动 (Sliding Down) |
| **`Shift + Tab`** | 候选词向上滚动 (Sliding Up) |
| **`Ctrl + Alt + G`** | 快速显示 / 隐藏 **传统候选窗** |
| **`Ctrl + Alt + H`** | 快速显示 / 隐藏 **卡片式候选词** |
| **`Ctrl + Alt + V`** | 循环切换自动粘贴模式 (Shift+Ins / Ctrl+V / ...) |
| **`Ctrl + Alt + P`** | 循环切换拼音预览模式 (无 / 拼音 / 汉字) |
| **`Caps Lock + Tab`** | 发送真实的 CapsLock 信号 (大写锁定) |

## 💻 命令行用法 (CLI Usage)

- `rust-ime`: 默认进入后台守护模式（前台实时打印候选词，便于 Debug）。
- `rust-ime --stop`: 安全停止进程并清理 PID。
- `rust-ime --restart`: 快速重载配置与词库。
- `rust-ime nihao`: 命令行即时转换拼音。

### 1. 格式支持
本系统支持 **JSON** (标准格式) 和 **YAML** (Rime 格式) 两种字典。

### 2. 第三方词库支持 (Special Thanks)
本项目深度集成并兼容 [**雾凇拼音 (Rime-ice)**](https://github.com/iDvel/rime-ice) 词库。

- **语义匹配**：支持从词库中提取英文定义实现“语义输入模式”（如输入 `you` 直达 `你`）。

## 🛠 技术架构与实现 (Technical Implementation)

### 1. 硬件级仿真 (Evdev & Uinput)
- **底层拦截**：通过 `evdev` 直接从 `/dev/input/event*` 拦截原始键盘事件。
- **虚拟硬件**：通过 `uinput` 创建虚拟 **HID 键盘设备**。将粘贴指令（如 `Shift+Insert`）模拟为真实的物理输入，不依赖桌面协议。

### 2. 现代 Wayland 适配 (Compositor Integration)
- **窗口感知**：基于外部命令实时感知的终端/GUI 自动适配方案（目前已简化为全通用的 `Shift+Insert` 逻辑）。
- **图层渲染**：候选框窗口基于 `gtk4-layer-shell` 开发，完美悬浮于 Wayland 图层之上。

### 3. 核心算法 (Core Algorithms)
- **空格破局法**：针对 Firefox 搜索框的自动补全干扰，独创 `Space + Backspace` 强制同步算法，确保首字母不被吞噬。
- **联想推理**：基于 **N-Gram** 语言模型，结合分段锁定逻辑，在保障灵活性的同时提供极高的输入确定性。
- **原子同步**：采用全量回退与重写的同步机制，具备天然的输入错误自愈能力。

1. **系统权限**：需加入 `input` 和 `uinput` 用户组。

## 🚀 性能与稳定性优化 (Stability & Performance)

### 1. 默认粘贴模式进化
- **Shift+Insert**：系统默认采用 `Shift+Insert` 作为上屏方式。这是 Linux 下最通用的粘贴快捷键，解决了 `Ctrl+V` 在特定环境下的冲突。

### 2. 极致响应
- **异步解耦**: 所有的外部检测、UI 渲染均与按键核心路径解耦，确保输入响应速度达到硬件级。
- **预加载机制**: 将拼音分词音节表等静态资源移至启动阶段加载，消除首次输入的“冷启动”卡顿感。
