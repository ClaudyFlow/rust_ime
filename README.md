# rust-IME 使用手册

这是一个专为 Linux 设计的高性能、现代化中文输入法引擎。采用 Rust 编写，深度适配 Wayland/COSMIC 桌面环境，提供极致的丝滑盲打体验。

---

## 💡 设计哲学与初衷 (Philosophy)

### 1. 为什么要做这个输入法？
拼音输入法最大的痛点在于 **“重码选择”**。即便全拼输入非常自然，但面对“里、力、理、利”等大量同音字时，选词过程极大地打断了思维流。

更深层的动力源于 **Linux Wayland 生态的输入困局**：
在现代 Linux（如 COSMIC 桌面）中，许多基于 Rust `iced` 库开发的原生应用对 `Fcitx5` 或 `IBus` 的支持极差。即使是系统自带的终端和文件管理器，也经常出现无法呼出输入法的情况。这通常是因为 Wayland 的 `text-input` 协议实现不统一导致的。

**rust-IME 的“降维打击”：**
- **原生通用性**：既然应用能收英文、能粘贴，那我们就把自己伪装成一个**真实键盘**。通过物理级的按键模拟方案，绕过复杂的 Wayland 协议逻辑，实现真正的“只要能打英文，就能打中文”。
- **英文辅码 (English Auxiliary)**：调用本能的英文知识实现**零重码盲打**。例如：你想打“泥”，只需输入 `niM`（M 代表英文单词 Mud）。

### 2. 行业锐评 (Hot Takes)
- **对比五笔**：学习曲线极其陡峭，且容易忘记拆解规则。
- **对比双拼**：只解决了“输入快”，没解决“选词慢”。
- **结论**：**全拼 + 英文辅码 + 简拼** 才是速度与直觉的完美平衡。

## 🛠 核心功能特性

### 1. 四声调过滤与智能纠错 (Tones)
利用数字键快速锁定声调，消除 80% 的重码：
- `7`: 一声 (ā) | `8`: 二声 (á) | `9`: 三声 (ǎ) | `0`: 四声 (à)
- **用法**：输入 `ni9` -> 直接过滤出“你”、“拟”等三声字。
- **智能纠错**：如果你输入的拼音拼写有误，程序会优先匹配符合声调的候选词。

### 2. 汉字学习模式 (Learning Mode)
- **原理**：在输入法闲置时，在按键回显位置（通常是角落）自动轮播展示汉字及其对应的英文释义。
- **扩展性**：支持加载**日语词库**（如 N1-N5 JSON），在打字之余利用碎片时间记忆假名或单词。

### 3. 高性能二进制架构
- **0 秒启动**：基于 `memmap2` 内存映射，无论词库多大，加载均为微秒级。
- **FST 索引**：O(1) 复杂度的前缀查找，绝无卡顿。

## 💻 命令行用法 (CLI Usage)

- `rust-ime`: 默认进入后台守护模式。
- `rust-ime --stop`: 安全停止进程并清理 PID。
- `rust-ime --restart`: 快速重载配置与词库。
- `rust-ime --install`: 一键安装桌面自启动。
- `rust-ime --train <dir>`: 递归扫描目录进行 N-gram 增量学习。
- `rust-ime nihao`: 命令行即时转换拼音。

## 📚 词库管理与扩展 (Dictionary Management)

### 1. 添加新词库
本系统支持 **JSON** (标准格式) 和 **YAML** (Rime 格式) 两种字典：
- 在 `dicts/` 目录下创建新文件夹（如 `dicts/my_dict/`）。
- 将你的 `.json` 或 `.dict.yaml` 文件放入该文件夹。

### 2. 编译词库 (Binary Compilation)
为了实现微秒级的搜索速度，新增或修改词库后需要编译为二进制索引：
```bash
cargo run --bin compile_dict
```
**特性**：
- **增量编译**：自动检测源文件变动，仅编译有修改的方案，极速跳过未变动词库（如大型 Rime 词库）。
- **结构化存储**：各方案数据独立存储在 `data/<方案名>/` 下，支持私有 N-gram 模型。

### 3. 第三方词库支持 (Special Thanks)
本项目深度集成并兼容 [**雾凇拼音 (Rime-ice)**](https://github.com/iDvel/rime-ice) 词库。
- **自动解析**：直接读取原生 YAML 格式，无需手动转换。
- **高性能转换**：通过内建编译器将百万级词条转换为高性能 FST 索引。

## 🛠 技术架构与实现 (Technical Implementation)

### 1. 硬件级仿真 (Evdev & Uinput)
- **底层拦截**：通过 `evdev` 直接从 `/dev/input/event*` 拦截原始键盘事件。这使得我们能 100% 接管全局快捷键，完全不受 Wayland 合成器对按键转发限制的影响。
- **虚拟硬件**：通过 `uinput` 在内核中创建一个虚拟的**标准 HID 键盘设备**。我们将处理后的按键序列（如 `Shift+Insert`）模拟为真实的物理输入发送给系统。这种方式不依赖任何特定桌面协议（如 DBus 或 Wayland 扩展），具有极高的生存能力。

### 2. 现代 Wayland 适配 (Compositor Integration)
- **窗口感知**：利用 `hyprctl`, `swaymsg` 和 `qdbus` (KDE) 实时获取当前活动窗口信息，实现“终端/GUI 应用”自动模式切换。
- **图层渲染**：候选框窗口基于 `gtk4-layer-shell` 开发，完美悬浮于 Wayland 图层之上，不干扰底层应用焦点。

### 3. 核心算法 (Core Algorithms)
- **拼音切分**：采用递归与贪心结合的切分算法，支持 `v` 替代 `ü` 及 ` `` ` 隔音符号。
- **联想推理**：基于 **N-Gram** 语言模型，结合用户历史输入频率，动态调整候选词权重。
- **差量渲染**：独创的预览差量算法，在退格时仅发送必要的删除信号，彻底消除在 Firefox 等重型应用中的闪烁与卡顿。

## 🚀 性能与稳定性优化 (Stability & Performance)

### 1. 默认粘贴模式进化
- **Shift+Insert**：系统默认采用 `Shift+Insert` 作为上屏方式。这是 Linux 下最通用的粘贴快捷键，完美适配终端与现代浏览器，解决了 `Ctrl+V` 在特定环境下的冲突。

### 2. 零 Panic 保证
- **安全解析**: 全面重构了 Trie 树与 N-gram 模型的二进制解析逻辑，引入边界检查，彻底消除因词库损坏导致的程序崩溃。
- **错误处理**: 移除了代码中所有的 `unwrap()` 调用，采用 Rust 惯用的 `Result` 和 `Option` 模式，确保在极端环境下依然能优雅处理错误。

### 3. 极致响应
- **异步解耦**: 将 GUI 消息发送与核心输入路径解耦。按键回显渲染在后台异步执行，确保不会因为 UI 绘制延迟导致输入法逻辑产生“连击”或“错位”。
- **预加载机制**: 将拼音分词音节表等静态资源移至启动阶段加载，消除首次输入的“冷启动”卡顿感。
