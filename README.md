# rust-IME 使用手册

这是一个专为 Linux 设计的高性能、现代化中文输入法引擎。采用 Rust 编写，深度适配 Wayland/COSMIC 桌面环境，提供极致的丝滑盲打体验。

---

## 🖥️ 桌面环境支持情况 (Compatibility)

由于使用了 `gtk4-layer-shell` 技术，本程序在不同桌面环境下的表现如下：

| 桌面环境 | 状态 | 说明 |
| :--- | :--- | :--- |
| **KDE Plasma (Wayland)** | ✅ 完美 | 核心测试环境，体验最佳。 |
| **COSMIC (Wayland)** | ✅ 完美 | 核心测试环境，深度适配。 |
| **Hyprland / Sway** | ✅ 支持 | 基于 wlroots 的合成器通常表现良好。 |
| **GNOME (Wayland)** | ⚠️ 受限 | 候选窗可能无法置顶或显示为普通窗口。 |
| **所有 X11 桌面** | ⚠️ 受限 | 界面层协议不支持，UI 体验可能下降。 |

> **提示**：即使 UI 无法显示，底层的“盲打”逻辑（注入、辅码、快捷键）依然在所有环境中有效。

## 🚀 核心灵魂特性 (Key Features)

### 1. 长韵母快捷输入 (Quick Rime / Tab 双快)
**借鉴小鹤音形设计，实现全拼速度的跨越。**
- **设计哲学**：保留全拼的拼写直觉，通过 `Tab` 键赋予按键“双重角色”。
- **操作方式**：按住 `Tab` 键的同时按下对应字母，瞬间注入长韵母。
- **当前配置 (根据你的 config.json)**：
    - `Tab + G` -> **eng**
    - `Tab + H` -> **ang**
    - `Tab + J` -> **iang**
    - `Tab + K` -> **uai**
    - `Tab + L` -> **uang**
    - `Tab + I` -> **ing**
    - `Tab + O` -> **ong**
    - `Tab + P` -> **iong**
    - `Tab + N` -> **iao**
    - `Tab + M` -> **ian**
    - `Tab + U` -> **uan**
- **优势**：大幅减少击键次数，比传统双拼更易上手，比纯全拼快一倍以上。

### 2. 英文辅码：零重码精准筛选 (English Auxiliary Code)
**调用本能的英文知识实现零重码盲打。**
- **设计哲学**：拼音解决“音”，英文辅码解决“义”。
- **操作方式**：输入拼音后，直接输入大写字母或单词片段。
    - **首字母匹配**：`liI` -> 定位 `里` (Inside)，排除 `力` (Power)。
    - **包含匹配**：`huaFlo` -> 匹配 `花` (Flower)。
- **智能联动**：支持与数字索引、空格联动，实现“精准打击”。

### 3. 智能防呆模式 (Anti-Typo)
**禁止输入不存在的拼音，让输入流永不断裂。**
- **逻辑**：实时校验当前片段。如果输入了错误的组合（如 `gog`），非法按键将被自动拦截。
- **反馈**：拦截发生时，系统会播放轻微的错误提示音（可配置开关），提供即时的触觉/听觉反馈。
- **辅码兼容**：智能识别辅码输入，一旦出现大写字母，防呆逻辑自动让位。

## 💡 设计哲学与初衷 (Philosophy)

### 1. 为什么要做这个输入法？
拼音输入法最大的痛点在于 **“重码选择”**。即便全拼输入非常自然，但面对大量同音字时，选词过程极大地打断了思维流。

**rust-IME 的“降维打击”：**
- **原生通用性**：伪装成真实键盘，通过物理级拦截 + 剪贴板中转，绕过复杂的 Wayland 协议逻辑。
- **环境要求**：高度依赖剪贴板服务。在 Wayland/X11 应用中表现极佳，但无法在无剪贴板的 TTY 原生模式下工作。

## 🛠 其他核心功能

### 1. 上屏模式切换 (Commit Mode)
- **词模式 (Word Mode)**：单击 `Space` 立即上屏首选。
- **长句模式 (Sentence Mode)**：单击 `Space` 锁定分词，双击 `Space` 或 `Enter` 上屏。
- **快捷切换**：按下 `Alt + Space` 快速切换。

### 2. 中英混输与转义 (/ Escape)
- **用法**：输入 `/` 后紧跟英文单词（如 `/rust`），该单词将以原始文本形式保留。

### 3. 全量中文标点转换
- 深度集成 `punctuation.json`，输入标点自动转中文并进入缓冲区预览。

### 4. 增强型拼音预览 (Enhanced Preview)
- **所见即所得**：在编辑框内直接显示输入的拼音和标点。
- **Tab 智能切词**：已迁移至 **左右方向键** 进行候选词循环选择。

### 5. 高性能二进制架构
- **0 秒启动**：基于 `memmap2` 内存映射。
- **自动增量编译**：启动时自动检测词库变动并更新二进制索引。
- **字典序优先**：移除复杂算法，顺序完全遵循字典原始定义。

## ⌨️ 默认快捷键 (Default Hotkeys)

| 快捷键 | 功能描述 |
| :--- | :--- |
| **`Caps Lock`** | 切换中 / 英文输入模式 |
| **`Ctrl + Space`** | 切换中 / 英文输入模式 (备选) |
| **`Alt + Space`** | **核心：切换上屏模式 (词模式 / 长句模式)** |
| **`Enter`** | 确认预览词上屏 |
| **`Space`** | 上屏 (词模式) / 手动分词 (长句模式) |
| **`Delete`** | 物理级一键清空输入缓冲区 |
| **`← / →`** | **移动光标选择候选词** |
| **`Tab + Letter`** | **长韵母快捷注入 (Quick Rime)** |
| **`Ctrl + Alt + G`** | 显示 / 隐藏 **传统候选窗** |
| **`Ctrl + Alt + H`** | 显示 / 隐藏 **卡片式候选词** |
| **`Ctrl + Alt + K`** | 开启 / 关闭 **按键显示** |
| **`Ctrl + Alt + V`** | 切换自动粘贴模式 (Shift+Ins / Ctrl+V) |
| **`/ (斜杠)`** | 开启英文转义模式 |

## 👨‍💻 AI 使用心得

在使用 Gemini 开发这个项目的过程中，我有一些非常深刻（甚至带点痛苦）的体会：

### 1. 关于 Gemini 的“性格”
Gemini 并不是一个稳定的指令执行者。即便我在 `GEMINI.md` 中明确写下了规范，它依然表现得非常“随缘”——有时会严格遵守，有时却完全视而不见，感觉它的输出质量完全取决于它当下的“心情”。

### 2. 那些还没解决的“大山”
虽然目前程序可用，但三大核心问题其实一个都没有真正解决：
- **算法陷阱**：AI 曾花了一整天写了一大堆看似高端的算法（DAG、N-Gram、广度优先搜索），但我根本看不懂。最后实测发现，这些复杂算法的效果还不如最简单的逻辑。于是我又花了一整天，把 AI 写的这些“智能”算法全部删干净了。
- **Wayland 协议**：这更是一场拉锯战，同样是折腾一天加代码，第二天发现不行又全部删掉，原地踏步。
- **UI 界面**：目前只勉强解决了一半。说实话，如果不是我后来找了一个同样用 Rust 写的 GTK4 按键显示项目强行喂给 AI 参考，估计它到现在也写不出可用的界面逻辑。

### 3. 核心感悟：AI 的天花板是使用者的水平
我发现 AI 能发挥出的水平，很大程度上受限于使用者的认知边界。

这就好比：如果你自己不懂“质数”的概念，却让 AI 写一个筛选质数的算法，当 AI 逻辑写错把 `6` 当成质数时，你自己根本看不出来，这就会导致开发完全无法进行。

同理，因为我对 DAG 和 N-Gram 的底层概念理解不深，想让 AI 在这种专业领域写出好代码是非常困难的。对于有海量数据支撑的大众化问题，AI 表现惊人；但面对稍微专业、小众的技术挑战，**“人”的判断力依然是不可逾越的最后一道防线。**


## 📦 安装与发布 (Installation & Release)

### 1. 开发者编译安装
如果你已经克隆了源码，可以直接运行：
```bash
chmod +x install.sh
./install.sh
```
脚本会自动安装依赖、配置权限、编译 Release 版本的程序并自动生成二进制词库。

### 2. 构建发行包
运行以下命令生成一个可供他人使用的独立安装包：
```bash
./make_release.sh
```
这会生成 `rust-ime-linux-x64.tar.gz`。用户只需解压后运行 `./install.sh` 即可。

### 3. 系统权限说明
由于程序涉及底层按键拦截，安装脚本会自动尝试：
- 将当前用户加入 `input` 组。
- 创建 `/etc/udev/rules.d/99-rust-ime-uinput.rules` 以允许非 root 用户操作 `uinput`。
- **注意**：修改权限后，需**重启系统**或**注销重登**才能生效。
