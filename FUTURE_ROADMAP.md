# Rust-IME Future Roadmap

## 1. 跨平台支持
- [ ] **Wayland 原生协议集成**: 
    - 使用 `zwp_input_method_v2` 实现真正的内联预编辑和光标跟随。
    - 解决 `wayland-client` 0.31 的协议代码隔离与冲突问题。
- [ ] **Windows (TSF) 支持**:
    - 实现基于 Windows Text Services Framework 的 Host。
    - 探索使用 Win32 API 拦截按键作为兼容模式。

## 2. 核心功能增强
- [ ] **语义输入模式 (English-to-Chinese)**:
    *   实现直接输入英文单词匹配对应汉字（如输入 `I` 匹配 `我`，`you` 匹配 `你`）。
    *   解决拼音与英文单词的冲突排序逻辑。
    *   建立英文关键词反向索引以保证查询速度。
- [ ] **卡片式候选词界面 (Modern UI) 深度打磨**:
    *   优化卡片视觉细节（阴影、边距、字体颜色）。
    *   增加平滑过渡动画和自适应布局。
- [ ] **Vim 式长句编辑功能**:
    *   支持在未上屏的长拼音串中移动光标（h/l 或箭头）。
    *   实现按音节/单词删除功能（类似 Vim 的 dw）。
    *   设计非侵入式的“编辑模式”切换逻辑。
- [ ] **光标跟随 (KDE/Wayland 临时方案)**: 
    - 探索通过 DBus 实时获取 KWin 光标坐标来移动当前 GTK 候选窗。
- [ ] **云同步**: 
    - 支持用户词库的云端备份与同步。
- [ ] **动态皮肤**: 
    - 在 Web 配置中心增加主题编辑器，允许自定义候选框 CSS。

## 3. 性能优化
- [ ] **词库索引优化**: 
    - 进一步压缩 Trie 索引，提升冷启动速度。
- [ ] **N-Gram 模型微调**: 
    - 增加用户习惯学习频率，提高首选词准确率。
