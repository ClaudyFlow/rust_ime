# Rust-IME Future Roadmap

## 1. 稳定性与安全性 (Stability & Security)
- [ ] **异常退出恢复机制**:
    - 实现 panic 钩子，确保程序崩溃时自动释放 `evdev` 的 `grab` 锁，防止用户物理键盘锁定。
    - 增加“安全守护进程”模式，主进程崩溃后自动尝试重启。
- [ ] **Web 配置中心安全强化**:
    - 默认强制绑定到 `127.0.0.1`，禁止外网访问。
    - 为 `/api` 接口增加简单的访问 Token 校验。
- [ ] **按键状态机重构**:
    - 针对 `Shift` 锁死和 `Modifier` 竞态问题进行形式化逻辑验证，确保模拟按键永远不会让系统陷入非正常按键状态。

## 2. 跨平台支持 (Cross-Platform)
- [ ] **Wayland 原生协议集成**: 
    - 使用 `zwp_input_method_v2` 实现真正的内联预编辑和光标跟随（作为 evdev 模式的替代或补充）。
- [ ] **Windows (TSF) 支持**:
    - 实现基于 Windows Text Services Framework 的 Host，复用 Trie 引擎内核。

## 3. 核心功能增强 (Core Features)
- [ ] **简拼/首字母匹配 (Abbreviation Matching)**:
    - 支持只输入声母路径（如 `nh` -> `nihao`），大幅降低长词击键数。
    - 结合英文辅码实现极速精准筛选（如 `nhL` -> `你好` Love）。
- [ ] **动态调频与个人词库 (Dynamic Recency & User Dict)**:
    - 记录用户最近选中的词条，并动态提升其在候选列表中的权重。
    - 实现简单的 LRU 机制存储用户私有词汇。
- [ ] **上下文二元模型 (Bigram Re-ranking)**:
    - 引入基于词语搭配频率的重排序算法（如打完“提高”后，“效率”自动排第一）。
- [ ] **Vim 式长句编辑**:
    - 支持在未上屏的长拼音串中进行精准光标移动和按音节删除。
- [ ] **智能中英混输 (Zero-switch Hybrid)**:
    - 识别特定模式（首字母大写、特定长度、包含特殊符号等）自动提供英文候选，无需手动切库。
- [ ] **快捷指令系统 (Slash Commands)**:
    - 支持 `/date`、`/time` 等指令直接生成动态内容。

## 4. 进阶体验 (Advanced Experience)
- [ ] **模糊音处理 (Fuzzy Pinyin)**:
    - 提供可选的模糊映射（如 `s/sh`、`z/zh`、`n/l`），提高高速打字下的容错率。
- [ ] **打字效率分析**:
    - 在 Web UI 增加 WPM（每分钟字数）实时曲线、误打率统计。
    - 生成用户专属的按键热力图。
- [ ] **极轻量 UI 探索**:
    - 考虑提供非 GTK 版本的渲染器（如基于 `tiny-skia` 或 `wgpu` 的原生渲染），进一步降低内存占用。
- [ ] **动态皮肤系统**:
    - 在 Web 配置中心提供可视化 CSS 编辑器，支持动态加载透明度、圆角和阴影效果。

## 5. 性能优化 (Performance)
- [ ] **大词库内存映射 (mmap)**:
    - 针对数百万词条的超大词库，使用 `mmap` 技术降低物理内存占用。
- [ ] **增量式实时编译**:
    - 在 Web UI 添加词库后，仅重新编译改动部分，实现秒级热更新。

## 6. 架构重构与解耦 (Architecture & Decoupling)
- [ ] **平台层彻底分离**:
    - 抽象“虚拟键盘事件”层，移除引擎对 `evdev` 或 Windows VK 的直接依赖。
    - 使得核心引擎 (`engine/`) 成为纯粹的逻辑黑盒。
- [ ] **GUI 驱动接口化**:
    - 定义 `CandidateWindow` trait，支持不同平台的渲染后端（Win32, GTK4, egui, tiny-skia）。
    - 允许主程序动态注入不同的 UI 实现。
- [ ] **标准化通信协议**:
    - 使用 `JSON-RPC` 或 `Bincode` 定义统一的跨进程通信协议。
    - 为未来的“远程输入法”或“多端同步”打下基础。
- [ ] **中心化配置观察者**:
    - 实现配置更新的广播机制，让所有模块（UI, TSF, Engine）实时响应 Web 端的修改。
- [ ] **显式状态机管理**:
    - 引入状态模式管理复杂的输入模式（导航、筛选、方案切换），减少热键冲突和逻辑死角。
